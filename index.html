Hackathon Guidelines

  • 👥 Team Size: 3 to 5 participants. Form teams before or at the event. Interdisciplinary teams encouraged.
  • 💻 Requirements: Projects must be built during the hackathon. Java Swing is required. Any tech stack may be integrated. Include a demo or presentation.
  • ⏱️ Time: 5-hour coding sprint with regular mentor check-ins.
  • 🧠 Judging: 40% Innovation, 30% Technical Complexity, 20% UX/UI, 10% Practical Impact.
  • 📜 Code of Conduct: Respect all, no harassment. Use open-source licenses.
  • 💡 Tips: Solve a real-world problem. Build a working prototype. Prepare a compelling pitch.
  • 📤 Submissions: GitHub repo link, 3-min demo video, README with setup instructions.
